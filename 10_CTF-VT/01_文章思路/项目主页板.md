# 🚀 Research Plan: Coarse-to-Fine Visuo-Tactile Fusion for Precision Assembly

## 1. 核心理念 (Core Idea)

**目标 (Goal):** 解决精密装配（Peg-in-the-hole）任务中，最后紧密装配时刻，末端执行器和物体遮挡视觉，导致状态估计失效的问题。
**方法 (Method):** 模仿人类操作逻辑，采用 **"Coarse-to-Fine"** (由粗到细) 的分层策略：
1.  **Coarse:** 利用 [[VLA]] (Vision-Language-Action) 模型进行语义理解和长程规划，到达操作空间。
2.  **Fine:** 利用 **视触觉融合 (Visuo-Tactile Fusion)** 的 [[Diffusion Policy]] 处理局部遮挡下的微调与插入。

> [!TIP] Innovation Point (IROS 投稿卖点)
> 传统的 VLA 缺乏几何精度，传统的 BC (Behavior Cloning) 缺乏泛化且受限于遮挡。本工作提出了一种 **多模态切换架构**，首次在 [[LeRobot]] 框架下验证了触觉图像 (Tactile Image) 在 Diffusion Policy 中的显式引入能显著提升遮挡下的装配成功率。


## 2. 系统架构 (System Architecture)

### 2.1 整体控制流

```mermaid
graph TD
    A[User Instruction: 'Insert cylinder to hole'] --> B(Global Planner: VLA Model);
    B -->|Output: Coarse Pose / Hover| C{Distance < Threshold?};
    C -- No --> B;
    C -- Yes/Occluded --> D(Local Controller: Tactile-Diffusion);
    D -->|Input: RGB + Tactile + Force| E[Action Execution];
    E --> F{Contact/Success?};
    F -- No --> D;
    F -- Yes --> G[Task Done
```

### 2.2 模型设计 (Modified Diffusion Policy)

基于 [[LeRobot]] 框架进行魔改，核心在于 **Multi-Modal Encoder** 的设计。

- **输入空间 (Observation Space):**
    
    - $I_{rgb}$: 外部/腕部相机 (Realsense)
        
    - $I_{tactile}$: 视触觉传感器图像 (Xsense style)
        
    - $F_{force}$: 6-DoF 力/力矩数据：来自触觉传感器解算
        
    - $q_{proprio}$: 机械臂关节状态
        
- **网络结构:**
    
    - **Vision Enc:** ResNet-18 / ViT
        
    - **Tactile Enc:** EfficientNet-B0 (专门提取纹理与形变特征)
        
    - **Fusion:** Concat -> MLP -> Diffusion Head (U-Net)
## 3. 实验计划 (Experiments)

### 3.1 任务设定

- **场景:** 桌面级 Peg-in-the-hole (圆柱、方块等不同形状)。
    
- **难点:** 设置强遮挡场景 (手完全挡住孔)，且孔位存在 $\pm 5mm$ 的随机误差。
    

### 3.2 对比基线 (Baselines)

1. **Vision-Only (BC/Diffusion):** 仅使用 RGB。预期结果：在遮挡时刻容易插歪或产生剧烈碰撞。
    
2. **Blind-Tactile:** 仅使用触觉。预期结果：搜索效率极低，难以找到初始孔位。
    
3. **Ours (CTF-VT):** 视觉引导 + 触觉修正。
    

### 3.3 评估指标 (Metrics)

- **Success Rate (SR):** 成功插入的概率。
    
- **Time to Completion:** 完成时间。

## 4. 实施路线图 (Roadmap)

## 4. 实施路线图 (Roadmap)

- [ ] **Phase 1: Realman+LeRobot HIL 打底**
    
    - [ ] 基于当前 `gym_manipulator_realman` 管线，稳定跑通 "Realman + Realsense + VR 遥操作" 的 HIL-SERL 录制流程（含 episode 边界与 SUCCESS/FAILURE 标签）。
        
    - [ ] 在简单 Peg-in-the-hole 场景下，录制一小批仅含 `RGB + Robot State + VR 动作` 的演示数据，验证数据集结构、`action/reward/done` 语义正确。
        
    - [ ] 评估控制环路的实际频率（目标 10Hz 左右），确认对 Diffusion Policy 在线控制是否足够稳定。
        
- [ ] **Phase 2: 视觉基线与任务定型**
    
    - [ ] 固化实验场景：确定 Peg 和 Hole 几何、公差范围、相机视角与照明，录制用于论文的 vision-only 演示数据集。
        
    - [ ] 在 [[LeRobot]] 中配置一个 "Vision-Only Diffusion Policy"（输入 `RGB + q_proprio`），先在 Realman 上实现稳定的视觉基线策略与评估流程。
    - [ ] 完成 Peg-in-the-hole 任务的成功率/时间等评估脚本，为后续对比实验打好基线。 
        
- [x] **Phase 3: 触觉与力传感器集成 (Hardware Bring-up)**
    
    - [x] 搭建 Xsense 触觉硬件到 Realman 末端执行器的机械与电气连接，获得稳定的触觉图像与 6-DoF 力/力矩输出。
        
    - [x] 编写/适配 ROS2 节点或 bridge，将 `Tactile Image + Force/Torque` 以带时间戳的消息形式发布，并与 Realman/相机时钟进行初步同步与校准。
        
    - [x] 标定触觉坐标系与末端执行器坐标系之间的外参，保证触觉特征可以和机器人状态在同一几何空间中解释。
        
- [ ] **Phase 4: 多模态数据管线 (Data Pipeline)**
    
    - [ ] 在现有 HIL-SERL 录制管线基础上，扩展 [[LeRobot]] 的 Dataset features，增加 `observation.tactile_image`、`observation.force` 等多模态观测键，并保证与 `RGB`、`Robot State` 在时间上的对齐精度。
        
    - [x] 编写数据可视化与对齐检查脚本（如：时间轴上同时回放 RGB、触觉图像和力曲线），用于验证同步质量和接触事件标注的合理性。
        
    - [ ] 明确数据采集 protocol：包括触碰、滑动、插入失败再修正等多种接触模式，为 Diffusion Policy 提供足够多样的 fine control 数据。 
        
- [ ] **Phase 5: 多模态 Diffusion Policy 与消融实验**
    
    - [ ] 在 [[LeRobot]] 现有 Diffusion Policy 实现上，增加触觉分支 Encoder（`I_tactile` + 力/力矩），设计与视觉特征的融合方式（Concat / Cross-Attn），并在配置文件中暴露可调的融合超参数。
        
    - [ ] 先离线训练/对比以下三种策略：    
        1. Vision-Only：`RGB + q_proprio`（Phase 2 基线）。
        2. Blind-Tactile：固定由视觉规划出的 coarse pose，仅用触觉 + 力反馈进行局部搜索与插入。
        3. Ours (Coarse-to-Fine VT)：视觉提供 coarse hover pose，触觉分支在接触后接管进行微调与插入。
        
    - [ ] 在 Realman 真机上复现上述三种策略的对比实验，重点观察：遮挡发生时的行为差异、接触力峰值、安全性以及成功率提升幅度。
        
- [ ] **Phase 6: 论文撰写与打磨**
    
    - [ ] 根据实验结果整理图表（成功率曲线、接触力曲线、典型可视化轨迹），撰写 IROS 草稿，并突出 "LeRobot + Realman + 触觉" 的工程复现价值。
        
    - [ ] 完成代码与数据集的整理与开源计划说明，方便 reviewers 和社区在 [[LeRobot]] 上复现本文方法。
